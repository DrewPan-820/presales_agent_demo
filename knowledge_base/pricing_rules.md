# 报价与选型规则（Pricing & Selection Rules）

> 本文件作为架构师角色的辅助知识，注入到 `architect.py` 的 system prompt 中。

## 1. 选型主逻辑

按"监控点位数"分档推荐边缘/中心硬件：

| 点位数 | 推荐硬件 | 备注 |
|--------|---------|------|
| 1–4 路 | `EdgeBox-S` × N（N = ceil(点位 / 4)） | 小型工位 |
| 5–8 路 | `EdgeBox-M` × N（N = ceil(点位 / 8)） | 中型产线 |
| 9–16 路 | `EdgeBox-L` × N 或 `EdgeBox-M` 多台 | 大型车间，按密度选型 |
| 16–32 路 | `CenterServer-32` 或 `EdgeBox-L` 多台 + `EdgeManager-20` | 看是否要集中部署 |
| 32+ 路 | `CenterServer-64`，或 `CenterServer-32` 多台 + `EdgeManager-100` | 大型工厂/多厂区 |

## 2. 必选附加项

- **推理盒管理终端**：当推理盒 ≥ 3 台时，必须配 `EdgeManager-20`（≤20 台）或 `EdgeManager-100`（>20 台）。
- **训练机 / 推训一体**：客户明确表达"现场模型迭代""数据不出厂""持续优化"等需求时，增配 `TrainBox-Pro` / `TrainBox-Ultra`，或直接选 `CenterServer` 推训一体方案。
- **实施服务费**：每个项目必含。≤20 路用"标准实施服务"，>20 路用"大型项目实施服务"。
- **算法定制服务**：客户场景明显非通用 SOP（如装配特殊动作识别），增加"算法定制服务"。
- **年度运维服务**：建议客户首年附加。

## 3. 报价输出要求

- 必须以 Markdown 表格形式列出：产品型号、用途、数量、单价、小计。
- 单价与小计统一以"万元"为单位。
- 表格末尾必须有"合计"行。
- 报价表后必须附"补充说明"，对待定项给出处理建议（例如带宽未确认 → 建议现场勘查）。
